# PawsMatch Asset Generation

This project generates artificial mock data (assets) for the PawsMatch app. PawsMatch is a web application where users can "Like" or "Pass" on dog profiles for adoption.

## Project Structure
- `main.py`: The entry point script that connects to the Google Gemini API to orchestrate data generation.
- `.env`/`.env.example`: Store environment variables like `GEMINI_API_KEY`.
- `pets.json`: Once generated, this file will contain 50 JSON-formatted dog profiles (with IDs, names, and 3-line Spanish bios).

## Stack Used
- **Python**: Core programming language.
- **[uv](https://github.com/astral-sh/uv)**: Used for extremely fast Python project initialization, dependency management, and virtualization.
- **google-genai**: Official Python SDK for interacting safely and ergonomically with Gemini.
- **Pydantic**: Used for robust JSON schema enforcement during structured output generation.
- **Gemini (`gemini-3-flash-preview`)**: LLM powering the asset generation.

## Data Generation Process
1. Load environment variables.
2. Initialize and authenticate `genai.Client`.
3. Pass a Pydantic schema enforcing `{ "profiles": [ { "id", "name", "bio" } ] }` to ensure strictly valid JSON outputs from Gemini.
4. Execute `gemini-3-flash-preview` to conjure high quality, nuanced mock data.
5. Save raw text data generated by Model to `pets.json` natively. 

## How to run
1. Set up dependencies: `uv sync` (or environment uses standard `uv` setups)
2. Create `.env` based on `.env.example` and put your `GEMINI_API_KEY` inside.
3. Run `uv run main.py`.
